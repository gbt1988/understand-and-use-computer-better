
# 🌀 Zsh 终端配置与环境变量完全指南

`.zshrc` 文件是一个脚本，每次你打开新的终端窗口时，它都会自动运行。它决定了你的终端外观、快捷键以及它如何查找软件。

## 一、 `.zshrc` 文件的构造
你的配置是分层构建的，从性能优化到视觉外观，再到个人工具链。

### 1. 性能与 UI 层 (即时响应)
在文件顶部，我们使用 **Powerlevel10k** 确保终端既美观又快速。
* **Instant Prompt (即时提示符):** 这是一个缓存脚本。它让终端在后台加载插件时，先瞬间显示出输入框，避免因插件过多导致的“启动卡顿”。
* **主题配置:** `ZSH_THEME` 变量告诉 Oh My Zsh 使用 Powerlevel10k 的视觉皮肤。

### 2. 框架与插件层
```zsh
export ZSH="$HOME/.oh-my-zsh"
plugins=(git)
source $ZSH/oh-my-zsh.sh
```
* **Oh My Zsh:** 一个管理配置、插件和主题的开源框架。
* **Plugins (插件):** 增加功能的脚本。例如 `git` 插件让你用 `ga` 代替 `git add`，极大提高效率。

### 3. 工具管理层 (Mise & Homebrew)
这些工具负责管理不同版本的软件（如 Python 3.10 和 3.12 切换）。
* **`eval`**: 该命令会将另一个程序的输出结果直接作为指令运行。例如 `eval "$(mise activate zsh)"` 允许终端在你进入不同项目文件夹时，自动切换对应的语言版本。

## 二、 深度理解环境变量 (Environment Variables)
环境变量是“键值对”（Key-Value pairs），充当计算机的全局记忆库。



### 核心规则：`$` 符号的妙用
在 Shell 脚本中，是否使用 `$` 符号意义完全不同：
* **不带 `$` (变量名):** 用于**定义**或**创建**变量。
    * `NAME="Claw"` （把 "Claw" 存入名为 NAME 的盒子）。
* **带 `$` (变量值):** 用于**读取**或**调用**内容。
    * `echo $NAME` （打开盒子查看里面的内容）。

### 常见系统变量对照表
| 变量 | 含义 | 示例 |
| :--- | :--- | :--- |
| `$HOME` | 当前用户的家目录 | `/Users/claw` |
| `$USER` | 当前登录的用户名 | `claw` |
| `$PWD` | 当前所在的文件夹路径 | `/Users/claw/projects` |
| `$SHELL` | 当前正在使用的 Shell 程序 | `/bin/zsh` |

## 三、 掌握 `$PATH`：系统的“导航图”
`$PATH` 是一个由冒号 `:` 分隔的目录列表。当你输入一个命令（如 `ls`）时，系统会从**左到右**依次查找这些文件夹。



### 为什么顺序很重要？
如果你在两个不同的文件夹里都有 `python` 程序，系统会运行在 `$PATH` 中靠左（优先级高）的那个。

### 高级技巧：路径去重
为了防止多次执行 `source ~/.zshrc` 导致路径重复，请在文件顶部加入：
```zsh
typeset -U path PATH
```
* **作用：** 自动过滤重复路径，保持 `$PATH` 变量简洁高效。

## 四、 Unix 文件系统层级逻辑
程序存放的位置决定了它的身份（是谁安装的，给谁用的）。



1.  **`/bin` & `/sbin`**: 系统核心工具（如 `ls`, `mkdir`）。这是计算机运行最基础的保障。
2.  **`/usr/bin`**: 操作系统预装的通用程序（如 `curl`, `python`）。
3.  **`/usr/local/bin`**: **Homebrew** 安装程序的地方。这是用户第三方软件的标准“安全区”。
4.  **`~/.local/bin`**: 你的**私人工具箱**。只有你能访问，通常优先级最高，用于存放你自己开发的脚本或 `mise` 管理的工具。

## 五、 安全实践：敏感信息脱敏
**永远不要**把 API 密钥（如 Claude Token）直接写在 `.zshrc` 中。

### 最佳方案：使用私密文件
1.  **创建文件:** `touch ~/.zsh_secrets`
2.  **锁定权限:** `chmod 600 ~/.zsh_secrets` (仅你自己可读写)。
3.  **存放密钥:** 在该文件内写入 `export TOKEN="xxx"`。
4.  **引用文件:** 在 `.zshrc` 末尾添加 `[[ -f ~/.zsh_secrets ]] && source ~/.zsh_secrets`。

## 六、 核心命令小结
* **`export`**: 将变量共享给当前终端启动的所有程序。
* **`alias`**: 为长命令创建绰号（如 `alias pon="..."`）。
* **`source`**: 立即在当前窗口应用配置文件的更改。
* **`env`**: 列出当前终端中所有生效的环境变量。

## 七、source和sh命令
source 的作用是读取并执行文件中的代码,source 命令的规则非常明确：它后面只能跟 文件路径，不能跟目录
⚠️ source 使用注意：
* 权限要求： 运行 sh script.sh 需要文件有“可执行权限”(x)，但 source script.sh 不需要可执行权限，只要你有“读”权限(r)即可。

* 空格敏感： source 和文件名之间必须有一个空格。

* 简写： 在 Zsh 和 Bash 中，. 等同于 source。例如 . ~/.zshrc 等于 source ~/.zshrc。

* 总结
source 后面跟的是包含 Shell 指令的文本文件路径。它就像是一个“阅读器”，把文件里的每一行字直接读进当前的终端里执行。
